# 마약류중독자관리시스템 — UI 기반 단계별 개발 가이드

> **기준**: `1.html` 시스템 개념도 · 현재 Next.js 프로토타입 분석 결과
> **목적**: 메인 UI의 각 모듈을 실제 업무화면으로 구현하기 위한 구체적 작업 목록

---

## 0. 현재 구현 현황 (베이스라인)

| 라우트 | 화면 | 구현 수준 | 비고 |
|--------|------|-----------|------|
| `/` | 개요(모듈 클릭) | ✅ 완료 | `OverviewSystem.tsx` |
| `/intake` | 접수/초기개입 | ✅ 데모 수준 | 5단계 Stepper, in-memory |
| `/cases` | 사례 목록 | ✅ 기본 | 목록·필터 |
| `/cases/[id]` | 케이스 상세 | ✅ 탭 5개 | 요약·타임라인·업무·모니터링·종결 |
| `/isp/[caseId]` | ISP 수립 | ✅ 기본 | 목표·개입 CRUD |
| `/integrations` | 연계관리 | ✅ 기본 | 커넥터 상태·로그 |
| `/stats` | 통계 대시보드 | ⚠️ 미흡 | 기초 KPI만, 차트 없음 |
| 대상자 관리 | 대상자 카드 | ❌ 미구현 | `/subjects` 없음 |
| 센터관리 | 센터/권역 | ❌ 미구현 | `/centers` 없음 |
| 권한관리 | RBAC/로그인 | ❌ 미구현 | 인증 없음 |
| 업무지원 | 서식/공지 | ❌ 미구현 | `/support` 없음 |
| 마감관리 | 월/분기/연 | ❌ 미구현 | `/closing` 없음 |

**공통 미구현**: DB 연동(전체 in-memory), 파일 첨부, 인쇄/출력, 감사로그

## Step 1. 대상자 관리 (Master) — 핵심 데이터 엔티티

> **목표**: 모든 업무의 중심이 되는 대상자 카드 CRUD 구현
> **1.html 위치**: `대상자 관리` 섹션 (마약류 중독자 · 가족 및 보호자 · 회복지원가)

### 1-1. 대상자 카드 (`/subjects`)

**라우트**:
```text
/subjects             → 대상자 목록 (검색·필터)
/subjects/new         → 신규 등록
/subjects/[id]        → 대상자 상세 (탭: 기본정보·가족·이력·케이스)
/subjects/[id]/edit   → 정보 수정
```

**타입 정의 추가** (`lib/types.ts`):
```typescript
export type Subject = {
  id: string;              // 내부 ID (UUID)
  caseNo: string;          // 관리번호 (예: SUBJ-000123)
  alias: string;           // 가명 (개인정보 보호)
  gender: "M" | "F";
  birthYear: number;
  region: string;
  drugType: string[];      // 마약 유형 (필로폰/대마/향정 등)
  entryRoute: "1342" | "LEGAL" | "SELF" | "AGENCY";
  registeredAt: string;
  status: "ACTIVE" | "MONITORING" | "CLOSED";
  familyIds: string[];     // 가족/보호자 ID 목록
  supporterIds: string[];  // 회복지원가 ID 목록
  caseIds: string[];       // 연결된 케이스 ID 목록
};
```

**화면 구현 항목**:
- [ ] 대상자 목록 테이블 (검색: 가명/관리번호/지역, 필터: 상태/마약유형)
- [ ] 신규 등록 폼 (가명, 성별, 출생연도, 지역, 유입경로, 마약유형 멀티선택)
- [ ] 개인정보 마스킹 표시 (이름 `홍*동`, 생년 `19**`)
- [ ] 대상자 상세 탭 구성:
  - **기본정보**: 등록일, 현재상태, 마약유형, 유입경로
  - **가족/보호자**: 관계 목록, 긴급연락처
  - **케이스 이력**: 연결된 `/cases` 목록
  - **회복지원가**: 담당 회복지원가 배정 현황

### 1-2. 가족 및 보호자 관리

**라우트**: `/subjects/[id]` 탭 내부 또는 `/family`

**구현 항목**:
- [ ] 가족/보호자 추가 폼 (관계, 연락처, 동거여부)
- [ ] 관계도 간단 표시 (텍스트 기반: 배우자 / 부모 / 자녀)
- [ ] 보호자 동의서 첨부 기능

### 1-3. 회복지원가 관리

**라우트**: `/supporters`

**구현 항목**:
