## 1. 프로젝트 개요

### 1-1. 시스템 목적

마약류중독자관리시스템은 마약류 중독자의 재활·치료 지원 업무를 전산화하여 중독자 관리 기관(센터)의 업무 효율을 높이고, 국가 통계 보고 체계를 표준화하는 시스템이다.

### 1-2. 주요 이해관계자

| 구분 | 역할 |
| --- | --- |
| 중독자재활센터 (일선 기관) | 대상자 접수·상담·사례관리 담당자 |
| 지역 권역 센터 | 권역 내 기관 통합 관리 |
| 마약류대책협의회 / 정부 부처 | 국가통계 보고 수요자 |
| 1342 콜센터 | 전화상담 접수·연계 |
| 시스템 관리자 | 권한·인프라 관리 |

### 1-3. 범위

- 마약류 중독자 접수부터 사례종결까지 전 주기 관리
- 재활교육·초기개입, 사례관리, 통계관리 핵심 3대 기능
- 대상자 관리 마스터, 공통 운영 기능, 인프라 연계

## 6. 데이터 모델 설계

### 6-1. 핵심 엔티티

```text
Subject (대상자)
├── id: string
├── name: string
├── residentNo: string (주민등록번호, 암호화)
├── drugHistory: DrugRecord[]
├── diagnosis: DiagnosisRecord[]
├── centerId: string → Center.id
├── assignedWorkerId: string → Worker.id
└── status: 'ACTIVE' | 'CLOSED' | 'MONITORING'

Case (사례)
├── id: string
├── subjectId: string → Subject.id
├── openedAt: Date
├── closedAt?: Date
├── closeReason?: string
├── events: CaseEvent[]
├── tasks: CaseTask[]
└── ispPlan?: ISPPlan

CaseEvent (타임라인 이벤트)
├── id: string
├── caseId: string
├── type: 'COUNSEL' | 'EDUCATION' | 'OUTREACH' | 'REFERRAL' | 'MONITOR'
├── occurredAt: Date
├── content: string
└── worker: string

ISPPlan (개인서비스계획)
├── id: string
├── caseId: string
├── goals: ISPGoal[]
├── interventions: ISPIntervention[]
├── reviewCycle: number (일)
├── nextReviewAt: Date
└── crisisPlan: string

Center (기관/센터)
├── id: string
├── name: string
├── region: string (권역)
├── contact: string
└── code: string

User (시스템 사용자)
├── id: string
├── username: string
├── passwordHash: string
├── role: Role
├── centerId: string → Center.id
└── isActive: boolean

Role (역할)
├── id: string
├── name: 'ADMIN' | 'CENTER_MANAGER' | 'CASE_WORKER' | 'VIEWER'
└── permissions: Permission[]

IntakeQueue (접수 큐)
├── id: string
├── channelType: 'CALL_1342' | 'WALK_IN' | 'REFERRAL' | 'ORDER'
├── receivedAt: Date
├── status: 'QUEUE' | 'SCREEN' | 'CONSENT' | 'ASSIGN' | 'CONVERTED'
└── convertedCaseId?: string → Case.id
```

### 6-2. DB 관계도 (간략)

```text
User ──── Center ──── Subject ──── Case ──── CaseEvent
                                    │
                                    └──── ISPPlan ──── ISPGoal
                                                  └──── ISPIntervention
```

## 8. API 설계 (백엔드 전환 시)

```text
POST /api/auth/login               로그인 (username + password)
```

```text
POST   /api/subjects/:id/family    가족 등록
```

```text
GET    /api/cases                  사례 목록
```
